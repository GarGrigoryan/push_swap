## Milestone Summary

**Key Decisions:**

- bench field placed last in t_ops — zeroed by ft_memset in init_ops, no init_ops changes
- Guard write() with `if (!ops->bench)` inline rather than a wrapper — norminette-safe, no signature changes
- count_only field added same way as bench — consistent pattern
- print_count() added to bench.c (not a new file) — keeps output helpers together
- Deleted compute_disorder duplicate from utils.c rather than commenting — no dead code in 42 submissions
- ft_strncmp n=13 for `--count-only` (length 12 + 1) — established codebase convention for prefix-match safety

**Issues Resolved:**

- Duplicate symbol linker error (compute_disorder in both utils.c and compute_disorder.c) — Phase 3
- Stale find_min_position prototype with no definition anywhere — Phase 3
- stdout not empty during bench mode — Phase 1 plan 02 gap closure

**Technical Debt Incurred:**

- run() in main.c has 8 variables — exceeds norminette 5-variable limit (deferred to norminette session)
- Prototype alignment in push_swap.h partially improved but some cosmetic items remain (IN-02 partially addressed)
- Makefile missing header dependency guard (IN-03 from code review, deferred)
- Commented pseudocode in compute_disorder.c (IN-01, deferred)
